What Does ‚ÄúFeature‚Äù Mean in Data Science?#

In data science and machine learning, a feature is just a fancy word for: A column in your table that describes something about your data.

  • Number of bedrooms üõèÔ∏è

Features of Special Format

In other fields, ‚Äúfeatures‚Äù can look very different:

  • üñºÔ∏è In computer vision:
    Each pixel can be considered a feature ‚Äî or features might be higher-level elements like color patterns, shapes, or edges.

  • üß† In NLP (Natural Language Processing):
    Features could include word counts, sentence length, or embeddings from a language model.

    For example, in Transformers, each token is represented as a 512-dimensional vector.

  • üßä In 3D data:
    Features might be points in a point cloud, meshes, or implicit surfaces ‚Äî used in fields like LiDAR, AR/VR, or 3D modeling & generation.

  • üé• In 4D data (e.g., video):
    Each frame can be treated as a feature, and you might also extract features within a frame (like objects), or track features across time (motion, trajectory, etc.).

  • üó∫Ô∏è In spatial data (our focus today!):
    A feature is a real-world geographic object ‚Äî something that has both location and descriptive attributes.

    Think of roads, buildings, rivers, or tram stops ‚Äî all stored with geometry and fields in shapefiles or geodatabases.

And just like in other fields, we still care about their properties ‚Äî but now they‚Äôre stored in attribute tables.

Take it easy ‚Äî imagine you‚Äôre the computer. For every item in a dataset, the things you store or know about it are called features. Features can be:

  • Built-in (directly collected from raw data)

  • Extracted, filtered, or converted from original information (e.g., converting time to day of week, PCA)

  • Learned through machine learning or deep learning models ‚Äî though these learned features can sometimes be a bit mysterious ü§ñ‚ú®

What Are Feature and what are Fields in a Shapefile?#

In this exercise, a feature specifically refers to a geometry object ‚Äî such as a point, line, or polygon ‚Äî representing a real-world location. Each feature typically consists of two parts:

  • Geometry: Describes where it is, including spatial shape and location. Also includes auxiliary information like the Coordinate Reference System (CRS).

  • Attribute Fields: Describe what it is ‚Äî these are the additional properties or metadata stored in a table (e.g., name, type, status, ID, etc.).

If you read or import shapefiles (or other spatial formats) in your program, you‚Äôll notice that all the information ‚Äî including geometry-related data ‚Äî is stored as attributes. So in a way, everything becomes a feature describing the data ‚Äî whether it‚Äôs where something is (geometry) or what it is (attributes). From the program‚Äôs perspective, it‚Äôs just a structured collection of information.

‚ö†Ô∏è Reminders

Always check the CRS (coordinate reference system) when working with multiple layers or datasets ‚Äî mismatched CRS is a common source of confusion!

What makes geoinformation special?#

In the lecture, we mentioned three key points that make geospatial data different from other types of data:

  1. Spatial and Temporal Dependence Example: Traffic congestion in one district affects neighboring areas; deforestation in one region impacts climate patterns elsewhere.

  2. Spatial Heterogeneity Example: Rainfall patterns or land use can differ dramatically from one region to another.

  3. Place-Sensitive User Interest Example: Local air quality reports are much more relevant to nearby residents.

üî¨ Dive more

  • In probability theory(e.g. Bayesian Estimation), the assumption of I.I.D. (Independent and Identically Distributed) is fundamental.
    But due to spatial and temporal dependence, this assumption often fails in geospatial contexts.
    üëâ How can we address this? ‚Üí Use models that account for spatial autocorrelation rather than assuming independence.

  • We can only collect samples at specific locations ‚Äî e.g., we have limited and static observations about weather conditions.
    üëâ How can we estimate values in unsampled areas? ‚Üí Use spatial interpolation techniques such as Kriging or IDW.

  • How can we quantify spatial heterogeneity?
    üëâ Use variograms, local statistics (e.g., Local Moran‚Äôs I), or model non-stationarity with techniques like GWR (Geographically Weighted Regression).
